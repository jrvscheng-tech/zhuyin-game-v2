# 注音闖關 Constitution

## Core Principles

### I. 純前端架構
- 無後端、無 API 依賴
- 所有遊戲邏輯在瀏覽器中執行
- 不使用 localStorage 或 cookie
- 積分僅在 session 內保持

### II. 可測試性優先
- 所有遊戲邏輯必須可被 unit test 覆蓋
- 目標覆蓋率 ≥ 80%
- UI 與邏輯分離，方便測試

### III. 幼兒友善設計
- 目標年齡層：5-7 歲兒童
- 色彩鮮明、操作簡單
- 拖曳介面直覺易用
- 正確答案有趣味音效回饋

### IV. 技術棧
- Vite 作為建構工具
- Tailwind CSS 處理樣式
- Vitest 進行單元測試
- GitHub Actions CI/CD
- GitHub Pages 部署

### V. 隱私保護
- 不收集任何用戶資料
- 無追蹤、無分析
- 完全離線可用（靜態檔案）

## 遊戲設計規範

### 題型設計
- **圖像配對**：顯示圖片，拖曳到正確注音箱
- **注音拼字**：圖片搭配空格，拖曳注音卡填入
- **注音排序**：打亂的注音卡重新排序

### 關卡結構
每關包含：
- 圖像配對 × 3 題
- 注音拼字 × 2 題
- 注音排序 × 1 題

### 積分系統
- 僅在 session 內保持
- 答對得分、答錯不扣分
- 每題限時但不強制（輔助提示）

## Governance

所有開發必須遵守此 constitution。任何修改需更新文件並記錄原因。
